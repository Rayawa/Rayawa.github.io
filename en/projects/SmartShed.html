Performance Tuning

Minimized JSON Payload size (e.g., {"intensity": 450}), removed redundant nesting, reduced public network transmission latency and packet loss rate. Frontend uses a single-partition flat architecture, converging high-frequency concurrent pressure into single-node communication, reducing the instantaneous handshake burden on the Broker under high packet loss conditions.

Embedded Side — Hi3861

Based on OpenHarmony LiteOS-M microkernel, developed in pure C

Hardware Connections

  • GPIO 11 → Green LED (Status indicator / Grow light feedback)
  • GPIO 12 → Yellow LED (Fault alarm / Device activation indicator)
  • I2C → AHT20 Temperature & Humidity Sensor
  • ADC → Light sensor, Soil moisture sensor

Network & Protocol Stack

  • LiteOS-M STA mode connecting to 2.4GHz Wi-Fi
  • DHCP dynamic local IP assignment
  • LwIP protocol stack → BSD Socket → TCP persistent connection (1883)
  • MQTT Client: Paho MQTT C Client port

Device Logic

  • Collection Thread: 1000ms periodic timer, ADC / I2C sampling
  • Control Interrupt: MQTT receive callback → GPIO PWM control
  • Fan / Water Pump / Grow Light support 0~3 stepless speed regulation

Key Technical Challenges

@ObjectLink deep nested object observation failure: In MQTT async callbacks, @ObjectLink only monitors first-level properties of objects, causing sensor numbers not to change and sliders being locked. Creatively adopted shallow copy destructuring assignment (this.zone1 = { ...this.zone1 }) to forcefully break through the state machine, perfectly fixing this critical bug.
