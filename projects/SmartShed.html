数据上报 Topics

反向控制 Topics

风扇 Rayawa/fan_1 补光灯 Rayawa/led_1 水泵 Rayawa/water_pump_1

性能调优

精简 JSON Payload 体积（如 {"intensity": 450}），去除冗余嵌套，降低公网传输延时与丢包率。前端采用单分区平铺架构，将高频并发压力收敛为单节点通信，减轻 Broker 在公网高丢包环境下的瞬时握手负担。

嵌入式端 — Hi3861

基于 OpenHarmony LiteOS-M 微内核，纯 C 语言开发

硬件连接

  • GPIO 11 → 绿色 LED（状态指示 / 补光反馈）
  • GPIO 12 → 黄色 LED（故障报警 / 设备激活指示）
  • I2C → AHT20 温湿度传感器
  • ADC → 光照传感器、土壤湿度传感器

网络与协议栈

  • LiteOS-M STA 模式连接 2.4GHz Wi-Fi
  • DHCP 动态获取本地 IP
  • LwIP 协议栈 → BSD Socket → TCP 长连接 (1883)
  • MQTT 客户端：Paho MQTT C Client 移植

设备逻辑

  • 采集线程：1000ms 周期定时器，ADC / I2C 采样
  • 控制中断：MQTT 接收回调 → GPIO PWM 控制
  • 风扇 / 水泵 / 补光灯支持 0~3 档无级变速

应用端 — HarmonyOS NEXT

关键技术攻关

@ObjectLink 深层嵌套对象观察失效：MQTT 异步回调中 @ObjectLink 仅监听对象首层属性，导致传感器数字不跳变、滑块被死锁。创造性地采用浅拷贝解构赋值（this.zone1 = { ...this.zone1 }）强制击穿状态机，完美修复致命 Bug。
